
## 概述
自用的练手项目。

# 项目一：通用的RAG系统（SimpleRAGSystem.py）
- 基于Langchain和Langgraph构造的RAG（检索增强生成）系统
- 包含文档加载、文本分块、存入向量数据库、检索、生成步骤。

## 技术架构
- **大语言模型**：deepseek-v4-flash
- **Embedding模型**：text-embedding-v1
- **向量数据库**：FAISS
- **框架**：Langchain+Langgraph

## 工作流程
- 用户查询 -> 查询改写（多轮对话时） -> 向量检索 -> 上下文构建 -> 回答生成 -> 置信度评估 -> 输出结果

## 核心模块说明
### 1. DocumentProcessor（文档处理器）
- 文档加载与分块
- 向量化与 FAISS 存储
- 支持持久化保存/加载
### 2. Retriever（检索器）
- 相似度检索
- 支持返回相似度分数
### 3. Generator（生成器）
- 查询改写（多轮对话）
- 基于上下文的回答生成
- 置信度评估
### 4. RAGChain（主控制器）
- LangGraph状态图编排
- 完整RAG流程管理


# 项目二：多Agent的电商客服系统（MultiAgentSupport.py）
- 基于Langgraph构建的多Agent客服系统。通过一个意图识别Agent，将客户需求分发给各自领域的Agent处理。

## 功能特性
- **智能意图识别**：自动识别用户意图（订单服务、技术支持、产品咨询、转人工）
- **多Agent协作**：将不同需求分发给各自专业领域Agent协同工作
- **质量检查**：自动评估回复质量，低质量回复自动转人工
- **工具调用能力**：各 Agent 可调用专属工具获取实时信息
- **状态管理**：基于 LangGraph 的工作流状态管理
- **安全降级**：异常情况自动转人工客服

## 系统架构
用户输入 → 意图分类器 → 路由分发 → 专业Agent → 质量检查 → 响应输出

 → 置信度<0.5 质量不合格
 → 人工客服


### Agent 类型

| Agent | 职责 | 工具 |
|-------|------|------|
| 订单服务 Agent | 订单查询、物流跟踪、退换货 | `query_order`, `track_shipping` |
| 技术支持 Agent | 故障排查、使用帮助 | `search_faq` |
| 产品咨询 Agent | 产品介绍、推荐、价格查询 | `search_product`, `get_product_recommendations` |
| 人工客服 | 投诉处理、复杂问题 | - |

## 手表充电慢的可能原因及排查步骤

### 1️⃣ 检查充电设备
- **使用原装充电器**：请确认使用的是手表原装充电器和充电线，非原装配件可能导致充电速度变慢。
- **检查充电线/充电座**：查看充电线是否有破损、弯折或接触不良的情况。
- **清洁充电触点**：用干净的干布轻轻擦拭手表背面的充电触点和充电器接触面，灰尘或污垢会影响充电效率。

### 2️⃣ 检查充电环境
- **温度影响**：手表在**过高（>35°C）或过低（<0°C）** 的温度下充电，系统会自动降低充电速度以保护电池。建议在室温（15°C~25°C）下充电。
- **充电时使用手表**：如果边充电边使用手表（如开启GPS、屏幕常亮等），耗电速度可能接近充电速度，导致感觉充得很慢。

### 3️⃣ 软件与系统问题
- **重启手表**：尝试重启手表，有时后台进程异常会导致充电变慢。
- **检查系统更新**：确保手表系统已更新到最新版本。

### 4️⃣ 电池健康状态
- 如果手表使用时间较长（1-2年以上），电池可能自然老化，导致充电效率下降。

---

### 建议操作顺序：
1. ✅ 先清洁充电触点并重启手表
2. ✅ 换到室温环境充电
3. ✅ 充电时关闭手表或开启飞行模式
4. ✅ 尝试用其他USB口或充电头

如果以上方法都试过了仍然充电很慢，建议您**联系人工客服**进一步检测，可能是电池或充电模块需要维修。请问您的手表是什么型号？使用多久了？我可以帮您进一步分析。
